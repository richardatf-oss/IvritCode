What’s happening on this page?

This page lets you watch a tiny “machine” made from Hebrew letters work on a few numbers that you choose.

Each letter is treated like an instruction. The little program we run here is: 

א → ב → ג → ד → סבב

You don’t need to know math or coding to play with it. Think of it like this:

  • א gets the machine ready and “locks in” that this run belongs to this program.
  • ב adds your first two numbers together.
  • ג multiplies them.
  • ד stirs the numbers around.
  • סבב does a deeper “mix” step used in modern cryptography.

Nothing here touches any accounts, wallets, or personal data. It’s just numbers and letters running in your browser.

Try It: Let the Letters Run

You choose four starting numbers. The machine runs the letters on them and shows every step it took.

  1. Type any whole numbers you like in the boxes.
  2. Click Run IvritCode Program.
  3. Open “Show raw execution trace” to see, line by line, what the machine did.
Show raw execution trace
How to read a line:
t=3 – step number in the run
pc – which letter we are on in the program
op – the letter that ran at this step
regs=[R0,R1,R2,R3] – the four visible numbers
c, h, z – hidden numbers the machine uses internally (for cryptography and consistency).

If You’re Just Curious

At a simple level, IvritCode is a way of saying:

  • “What if the Aleph-Bet were a tiny machine?”
  • “What if a story was also a set of instructions?”
  • “What if we could watch letters move numbers around?”
  • No background knowledge required.
  • You cannot break anything by experimenting.
  • Every run is repeatable: same letters, same numbers, same result.

Later, this site will grow a visual playground where you can drag letters onto a line, press play, and see the machine animate its steps without looking at any code at all.

If You’re a Developer or Researcher

Under the hood, everything you see here is driven by a small virtual machine written in TypeScript:

  • Each letter in the demo has a precise definition in src/vm.ts.
  • Arithmetic is done in a finite field (see src/field.ts).
  • The deeper “mixing” step סבב uses a Poseidon-style round function (src/round.ts).

The VM is designed so its execution trace can be expressed as an AIR (Arithmetic Intermediate Representation) for zk-VM style proofs.

Letters Used in This Demo

This first version only uses a handful of letters. More will be added later, but these are enough to see the idea:

Letter Name What it does in the machine (plain language)
א Aleph Leaves your four numbers as they are, but secretly updates some hidden numbers so this run is tied to this program and moment in the code.
ב Bet Adds the first two numbers together and stores the result back into R0.
ג Gimel Multiplies the first two numbers and stores the result in R0.
ד Dalet Takes R0, R1, and R2 and stirs them together using a simple linear recipe. R3 is left alone.
סבב Sovev Runs a deeper mixing step over all four numbers plus one hidden number. This is the kind of step used inside modern hash functions.
